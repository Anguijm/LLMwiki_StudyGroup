# ingest-pdf v1 (orchestration doc)

**This is a documentation file, not an LLM prompt.**

The ingest.pdf Inngest function composes the following vendor interactions.
Any change to the pipeline shape lands here first, then flows into the
function's step implementations.

## Flow

1. `ingest.pdf.requested` event — carries `{job_id, idempotency_key, owner_id, cohort_id, storage_path}`.
2. `parse` step:
   - Magic-byte check on the uploaded file bytes.
   - Call `pdfparser.parse(storageSignedUrl)`.
   - If the parser returns `page_count=0` or every block has empty text, fail with `pdf_no_text_content`.
   - Concatenate text blocks into contiguous sections preserving headings.
3. `chunk` step:
   - Heading-aware chunking; target ~1.2k tokens per chunk.
   - **Hard cap `MAX_CHUNKS = 200`** — exceeding fails the job with `pdf_too_many_chunks` (no silent truncation).
4. `token_budget_reserve` step:
   - Estimate total tokens for remaining pipeline: `sum(chunk_tokens) * simplify_fanout + embed_tokens`.
   - Check `ingestion_jobs.reserved_tokens`; if non-null, skip (already reserved).
   - Atomically INCRBY the estimate into Upstash Tier B; persist the estimate to `ingestion_jobs.reserved_tokens`.
   - On overflow: fail with `token_budget_exhausted`, expose `resetsAt` to the UI.
5. `simplify` step:
   - Batch chunks up to 8 per Haiku call.
   - System prompt: `packages/prompts/simplifier/v1.md` with `cache_control: {type: 'ephemeral'}` on the stable prefix.
   - Each chunk wrapped in `<untrusted_content index="i">...</untrusted_content>`.
   - Output concatenated into `body_md`. If the batch returns `[[NO_TEXT_CONTENT]]`, treat as empty and surface `pdf_no_text_content`.
6. `embed` step:
   - If `body_md.length > VOYAGE_MAX_INPUT_CHARS`, fail with `embed_input_too_long` (no silent truncation).
   - Otherwise call `voyage.embed(body_md)`.
7. `persist` step:
   - Generate UUID client-side; build slug as `slugify(title) + '-' + short_hash(id, 6)` (fallback to longer hash on collision).
   - `INSERT ... ON CONFLICT (source_ingestion_id) DO NOTHING RETURNING id` so retries are no-ops.
8. `post-ingest.enqueue` step — emits `note.created.link` + `note.created.flashcards` (no-op stubs in v0).

## Failure paths

Every step catches its typed error (`pdf_*`, `embed_input_too_long`, `ai_response_shape_error`, `ai_request_timeout_error`, `token_budget_exhausted`, `ratelimit_unavailable`) and sets `ingestion_jobs.status='failed'` with `error: {kind, message, step}` before rethrowing.

The function-level `onFailure` hook runs on any terminal failure (including watchdog rescues) and:

1. Atomically nulls `reserved_tokens` + refunds the Upstash counter (only on non-null return).
2. Deletes `ingest/<job_id>.pdf` from Supabase Storage.
3. Emits cleanup metrics.
